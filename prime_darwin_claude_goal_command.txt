/goal Build Prime Darwin Production V0 in agent-darwin.

You are Claude Code acting as a strict senior implementation worker, tester, and scope guard.

Project:
agent-darwin

Mission:
Build Prime Darwin V0: a copied-repo evolutionary software factory that takes Mohit's raw brainstorm/master plan, creates plan/prompt/context variants, runs multiple isolated candidate repo copies through OpenCode SDK worker/evaluator lanes, applies GitSafe snapshots/tests/rollback, produces evidence-backed rankings, generates Poolside second-review prompts after every milestone, creates Codex/Opus audit artifacts, and never touches original repos or auto-promotes anything.

Current direction locked:
- OpenCode SDK is primary programmable model/tool hub.
- Poolside pool CLI is second-review lane after every milestone.
- Darwin owns GitSafe snapshots, candidate copies, scoring, memory, reports, and promotion recommendations.
- Codex remains strict code/test reviewer.
- Opus remains final audit/promotion advisor through audit prompt artifacts.
- Aider is optional later; do not fork Aider or depend on Aider internals in this chunk.
- LangGraph is later; do not add LangGraph until simple Python proof works.

Primary user workflow:
Mohit points Darwin at an already copied repo and a master plan.
Darwin preserves raw plan.
Darwin creates 3 plan versions, 2 prompt versions, and 2 context versions.
Darwin scores variants using OpenCode SDK structured JSON plus Darwin rules.
Darwin creates 4 candidate repo copies.
Darwin runs OpenCode SDK build/eval inside candidate copies.
Darwin uses GitSafe wrapper for snapshot, tests, commit/rollback.
Darwin creates Poolside pool review prompts after every milestone.
Darwin captures Codex review artifacts and Opus audit prompts.
Darwin ranks candidates and writes MORNING_REPORT.md.
Mohit manually promotes the winning copy.

Hard rules:
1. Work in small milestones/chunks only.
2. After each milestone, run tests and write SELF_AUDIT.md.
3. After each milestone, generate a POOLSIDE_REVIEW_PROMPT.md for second review.
4. Do not proceed past a failing milestone unless the fix is local, minimal, and retested.
5. Never touch original repo paths except reading explicitly provided current repo source if needed.
6. Never git push.
7. Never auto-merge.
8. Never store or print API keys.
9. Never read/write/copy .env or secrets.
10. No infinite loop in V0. Use max_generations and stop rules.
11. Smoke tests must use fake OpenCode/Poolside/Codex workers and must not call real APIs.
12. OpenCode SDK bridge must support fake mode first, then real mode behind env/config.
13. Poolside review must be generated as an artifact prompt; do not require Poolside to be installed for smoke tests.
14. Keep existing Darwin commands and smoke tests working.
